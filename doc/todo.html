Ewok 1.0/Wicket - Current Hit List

Feb 18th 2010

In general, these items are listed somewhat in order of priority...

ewok_installer

  • deploy web module as #web_app

ewok_deployment_srv

  • finish load/deploy logic (mostly done)
  • complete the "packaging" call for tpas (third-party applications)

ewok_admin (application)

  • Ajaxify the app
  • complete general admin tasks for the admin web interface
  • add development functionality to the admin web interface

esp

  • complete request delegation - the approach is still not 100% clear, but may become so when you've written a few more ewok applications...
  • complete ESP templates and esp:include

esp_ajax

  • define and implement ewok.js... should very likely use the Observer/Event Handler pattern (but beware crappy ids littering the html.
  • define and implement ubf.js, which will be an UBF codec implementation for javascript.

esp_html

  • provide a pretty-printer for html that would be used in development mode

ewok_http_srv

  • verify support for HTTP/1.0 in the codebase (main reason? NginX upstream / proxy-pass)

ewok_sdb

  • add query by attributes
  • add datasource interface/gen_server
  • update to latest AWS interface version

ewok_app

  • allow an "embedded" mode

ewok_sup

  • correctly register service instances by tcp port, e.g. ewok_http_srv_8080

ewok_session_srv

  • consider changing session server cleanup to use scheduled tasks

ewok_logging_srv

  • use scheduled tasks for log rollover

ewok_configuration

  • reconsider offering the original property file format, e.g. "ewok.http.port=8000", as well as the more correct OTP 'term' implementation -- (remember how you felt about SciTe config)

ewok_identity_srv

  • move realm access validation/control from ewok_http_srv into here? (or an ewok_security module?)
  • make authentication pluggable

general

  • move service APIs to own modules to clarify purpose improve the API naming and "protect" the underlying gen_servers from application developers (partly done)
  • consistently handle strings as binaries *throughout* the codebase (nearly done)
  • review logging throughout the codebase

ewok_uws

  • define and create the UBF web service interface/implementation
  • consider this as a basis for all AJAX calls, with XML Web Services implemented as an alternative facade to UWS

ewok_request_obj

  • chunked/multipart transfer (ref ewok_http_client impl of transfer-encoding)

ewok_response

  • chunked/multipart transfer
  • ranged requests

ewok_mail

  • improve smtp client for outgoing mail

ewok_http_client

  • improve http client to add post, range requests

ewok_data_srv

  • consider how best to provide pooled connections (where necessary)
  • figure out "generic metadata" access via ewok_datasource interface

ewok_geoip_srv

  • convert egeoip into an ewok_service
  • implement country lookup/gen_server (use mnesia/default ds)
